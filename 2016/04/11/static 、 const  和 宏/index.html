static 、 const 和 宏

@(iOS)

苹果不推荐大量使用宏定义，推荐使用const常量。但宏有个好处就是能定义函数。

static

修饰局部变量 : 

> - 1、局部变量只初始化一次，只占一块内存。
> - 2、不改变作用域，只是改变了周期，在方法结束时不会销毁。

修饰全局变量：

  • 1、全局变量的作用域仅限于本文件

const

被修饰的变量不能在后面被修改。

  • const修饰的 *p

    被const修饰的*p 只能被赋值一次，以后不能赋值，否则编译器报错

  • const修饰的p

    被const修饰的p只能存一次地址，以后再也不能其它存地址了，否则编译器报错

  • const在声明字符串的用法

static和const联合使用

  • static将一个全局变量变成局部变量
  • const将一个局部变量变成局部常量

static const 与 宏

相同点

  • 都不能再被修改
  • 一处修改，其它都改了

不同点:

  • static const修饰变量只有一份内存
  • 宏定义，只是简单的替换，每次使用都需要创建一份内存
  • const是在预编译，而宏是编译阶段。
  • 宏不会报编译错误，const 会报编译错误。

苹果建议尽量使用const。

总结：

使用static const修饰更加高效，在同一个文件内可以使用static const取代#define
